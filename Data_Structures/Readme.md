# Data Structures
A **Data Structure** is a way of organizing data in a computer so that it can be used efficiently.

## Foundational Terms of Data Structures
- **Implementation**: This provides the internal representaion of a data structure and the algorithms used in its operations.
- **Interface**: This defines the set of operations that a data structure supports.

**Abstract Data Type (ADT)**: It is a data type that defines the operations of a data structure without specifying its implementation.

## Characteristic of Data Structures
- **Correctness**: A data structure implememtation must satisfy the interface.
- **Time Complexity**: Running time of the operations a data structure performs must be small as possible.
- **Space Complexity**: Amount of memory used by a data structure must be small as possible.

## Classifications of Data Structures
- **Simple Data Structure**: A simple data structure is a data structure that is formed using primitive data types.
- **Compound Data Structure**: A compound data structure is a data structure that is formed using more than one simple data structures.
    - **Linear Data Structure**: A linear data structure is a data structure in which the elements are stored in a linear sequence in memory.
    - **Non-Linear Data Structure**: A non-linear data structure is a data structure in which the elements are stored in a non-linear sequence in memory.

## Operations on Data Structures
- **Insertion**: Adding an element to a data structure.
- **Deletion**: Removing an element from a data structure.
- **Search**: Finding an element in a data structure.
- **Update**: Modifying an element in a data structure.
- **Traverse**: Visiting each element in a data structure.
